# 纸条系统架构说明

## 目录结构

```
NoteLoaderModel/
├── SecretNoteCore.cs        # 纸条系统核心管理类
├── NoteDataManager.cs       # 纸条数据管理器
├── NoteItemManager.cs       # 纸条物品管理器
├── NoteActionExecutor.cs    # 纸条动作执行器
├── NoteCollectionManager.cs # 纸条收集管理器
├── NoteUIManager.cs         # 纸条界面管理器
├── NoteDisplayManager.cs    # 纸条显示管理器
├── ItemUseDetector.cs       # 物品使用检测器
├── ItemUseEventHandler.cs   # 物品使用事件处理器
├── ModEntry.cs              # 模组入口静态引用
```

## 核心组件说明

### 1. SecretNoteCore.cs

- **功能**：纸条系统的核心管理类，统一协调所有子模块
- **职责**：
  - 初始化和管理所有子模块
  - 处理资产请求事件
  - 应用Harmony补丁
  - 提供对子模块的访问接口
- **核心方法**：
  - `Register()`: 注册和初始化所有模块
  - `OnAssetRequested()`: 处理资产请求

### 2. NoteDataManager.cs

- **功能**：负责加载和管理纸条数据
- **职责**：
  - 从JSON文件加载纸条数据
  - 管理纸条ID映射
  - 处理资产请求，将纸条数据添加到游戏中
- **核心方法**：
  - `LoadNoteDatas()`: 加载纸条数据
  - `LoadIdMappings()`: 加载ID映射
  - `HandleAssetRequest()`: 处理资产请求
  - `GetOrAssignNoteId()`: 获取或分配纸条ID

### 3. NoteItemManager.cs

- **功能**：负责纸条物品的创建和使用
- **职责**：
  - 注册纸条物品使用规则
  - 处理纸条物品的使用事件
  - 为纸条创建物品数据
- **核心方法**：
  - `RegisterNoteUseRule()`: 注册纸条使用规则
  - `HandleNoteItemUse()`: 处理纸条物品使用
  - `HandleAssetRequest()`: 处理物品数据资产请求

### 4. NoteActionExecutor.cs

- **功能**：负责执行纸条相关动作
- **职责**：
  - 处理纸条附件（如物品奖励）
  - 执行触发动作（如解锁内容、添加任务）
- **核心方法**：
  - `TriggerNoteActions()`: 触发纸条动作
  - `HandleAttachments()`: 处理附件
  - `TriggerAction()`: 触发单个动作

### 5. NoteCollectionManager.cs

- **功能**：负责管理纸条收集状态
- **职责**：
  - 跟踪玩家已收集的纸条
  - 提供添加、检查、获取和移除纸条的方法
- **核心方法**：
  - `AddNote()`: 添加纸条到收集
  - `HasNote()`: 检查玩家是否已收集纸条
  - `GetNotes()`: 获取玩家已收集的所有纸条
  - `RemoveNote()`: 从收集中移除纸条

### 6. NoteUIManager.cs

- **功能**：负责管理纸条相关界面
- **职责**：
  - 应用Harmony补丁修改游戏界面
  - 在收集品页面显示纸条
  - 处理纸条点击事件
- **核心方法**：
  - `ApplyPatches()`: 应用Harmony补丁
  - `ShowSecretNote()`: 显示纸条内容
  - `CollectionsPage_ctor_Postfix()`: 修补收集品页面构造函数
  - `CollectionsPage_receiveLeftClick_Postfix()`: 修补收集品页面点击事件

### 7. NoteDisplayManager.cs

- **功能**：负责显示纸条内容
- **职责**：
  - 显示秘密纸条界面
  - 处理纸条背景和内容显示
- **核心方法**：
  - `ShowSecretNote()`: 显示秘密纸条

### 8. ItemUseDetector.cs

- **功能**：负责检测物品使用事件
- **职责**：
  - 监听物品使用事件
  - 执行注册的使用规则
- **核心方法**：
  - `RegisterUseRule()`: 注册物品使用规则
  - `OnButtonPressed()`: 处理按钮按下事件

### 9. ItemUseEventHandler.cs

- **功能**：物品使用事件处理器
- **职责**：
  - 定义物品使用规则和事件
  - 处理物品附件

### 10. ModEntry.cs

- **功能**：模组入口静态引用
- **职责**：
  - 存储IModHelper和IMonitor的静态引用
  - 在静态上下文中提供依赖访问
- **核心方法**：
  - `Initialize()`: 初始化静态引用

## 数据流和依赖关系

### 初始化流程

1. `SecretNoteCore.Register()` 被调用
2. 初始化 `ModEntry` 静态引用
3. 初始化所有子模块：
   - `NoteDataManager`
   - `NoteActionExecutor`
   - `NoteCollectionManager`
   - `NoteItemManager`
   - `NoteUIManager`
4. 注册资产请求事件
5. 应用Harmony补丁

### 纸条使用流程

1. 玩家右键点击纸条物品
2. `ItemUseDetector` 检测到物品使用
3. `NoteItemManager.HandleNoteItemUse()` 被调用
4. 查找对应的纸条数据
5. `NoteDisplayManager.ShowSecretNote()` 显示纸条
6. `NoteActionExecutor.TriggerNoteActions()` 执行纸条动作
7. `NoteCollectionManager.AddNote()` 将纸条添加到收集品列表

### 收集品页面流程

1. 玩家打开收集品页面
2. `NoteUIManager.CollectionsPage_ctor_Postfix()` 修补构造函数，添加纸条到页面
3. 玩家点击纸条
4. `NoteUIManager.CollectionsPage_receiveLeftClick_Postfix()` 修补点击事件，显示纸条内容

## 架构特点

1. **模块化设计**：每个组件职责清晰，便于维护和扩展
2. **依赖注入**：通过构造函数注入依赖，提高代码可测试性
3. **静态引用**：通过ModEntry在静态上下文中提供依赖访问
4. **Harmony补丁**：使用Harmony修改游戏界面，实现纸条在收集品页面的显示
5. **资产处理**：通过资产请求事件，将纸条数据和物品数据添加到游戏中

## 扩展建议

1. **添加配置系统**：允许玩家自定义纸条系统行为
2. **增加纸条类型**：支持不同类型的纸条（如任务纸条、剧情纸条等）
3. **添加本地化支持**：支持多语言纸条内容
4. **实现纸条解锁条件**：根据游戏进度或玩家行为解锁纸条
5. **增加纸条统计功能**：显示收集进度和成就

## 技术要点

- **Harmony补丁**：用于修改游戏界面，实现纸条在收集品页面的显示
- **资产加载**：通过SMAPI的资产请求事件，动态添加纸条数据和物品数据
- **事件系统**：使用SMAPI的事件系统，监听游戏事件
- **数据持久化**：使用玩家的modData存储纸条收集状态
- **模块化架构**：采用模块化设计，提高代码可维护性和扩展性
